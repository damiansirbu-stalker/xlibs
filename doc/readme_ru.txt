xlibs: Разделяемая библиотека утилит для моддинга STALKER Anomaly, автор Damian
Версия: 1.4.1
GitHub: https://github.com/damiansirbu-stalker/xlibs
Список изменений: https://github.com/damiansirbu-stalker/xlibs/blob/main/doc/changelog
English: https://github.com/damiansirbu-stalker/xlibs/blob/main/doc/readme.txt
Сообщить об ошибке: https://github.com/damiansirbu-stalker/xlibs/issues

xlibs -- набор инструментов моддера, покрывающий всё, что обычно нужно модам Anomaly: запросы сущностей, операции с отрядами, логика умных укрытий, работа с тайниками, логирование, профилирование, системы событий и структуры данных.

Дизайн API основан на реверс-инжиниринге движка X-Ray и внутренностей Anomaly, с перекрёстной проверкой по паттернам лучших моддеров как европейской, так и русской традиции моддинга STALKER. Каждая функция оборачивает причуды движка, защищает от nil и обрабатывает крайние случаи, которые иначе каждому моду пришлось бы решать самостоятельно.

Чистый Lua где возможно. Без зависимости от движка, если не необходимо. Без центрального загрузчика -- движок автозагружает скрипты при первом обращении. Просто вызовите xlog.get_logger() или xsquad.find_squads() -- и всё работает.

Возможности:

Сущности и мир:
  xcreature    Идентификация существ, проверки типов, итератор серверных объектов
  xsquad       Поиск отрядов, скриптовое управление, освобождение, погоня, итерация
  xsmart       Запросы умных укрытий, определение фракции, вместимость, прибытие, захват
  xlevel       Запросы уровней/карт, игровое время, названия локаций, валидация вершин
  xobject      Разрешение серверных объектов из любого типа входа, создание предметов офлайн
  xstash       Обнаружение тайников, ограбление, пополнение, фильтрация предметов
  xdata        Таблицы нескриптуемых NPC/отрядов (торговцы, механики, сюжетные персонажи)
  xconst       Константы-часовые движка X-Ray (невалидный ID сущности, невалидный ID вершины уровня)

Структуры данных:
  xtable       Фильтр, поиск, свёртка, клонирование, перемешивание, сортировка, бинарная вставка, мемоизация, блокировки
  xttltable    TTL-хранилище ключ-значение и скользящий оконный счётчик с автоистечением
  xmath        Броски вероятности, взвешенный выбор, случайная выборка, вариация
  xslice       Покадровая итерация массивов
  xstring      Строковая интерполяция с плейсхолдерами ${key}
  xtime        Аккумулятор игрового времени в секундах (обёртка над game_time())

Диагностика:
  xlog         Буферизованное файловое логирование с управлением сессиями и ротацией
  xprofiler    Профилирование кода с точностью до микросекунд через profile_timer движка
  xtrace       Генерация ID трассировки для отслеживания запросов между модулями
  xinspect     Глубокая инспекция таблиц и userdata, идентификация типов движка

Интеграция:
  xbus         Pub/sub шина событий с pcall-обёрткой доставки и диагностикой
  xevent       Рантаймовый хукинг функций для синтетических коллбэков
  xpda         КПК-сообщения и маркеры карты (отряды и сущности)
  xmcm         Геттеры и загрузчики конфигурации MCM с извлечением значений по умолчанию

Требования:
Anomaly 1.5.3

Установка (MO2):
1. Установите xlibs
2. Порядок загрузки не имеет значения (скрипты автозагружаются движком по требованию)

Удаление (MO2):
Отключите или удалите в MO2. Любой мод, зависящий от xlibs, перестанет работать.

Конфигурация:
Настройка не требуется. xlibs -- пассивная библиотека, загружаемая по требованию другими модами.

Совместимость:
Протестирована на ванильном Anomaly 1.5.3, Demonized main, Demonized MT и AOEngine (актуальные версии).
Чистая библиотека. Не модифицирует базовые скрипты, не регистрирует коллбэки, не запускает фоновую логику. Совместима со всем, включая GAMMA.

Разработка:
Написано на основе исходного кода X-Ray Monolith, исходного кода моддед-экзешников Demonized и распакованных gamedata Anomaly 1.5.3.
Паттерны кода и использование движка валидированы по работам авторитетных GAMMA-моддеров (Demonized, Vintar0, RavenAscendant, xcvb).
Код валидируется в реальном времени многоступенчатым конвейером: luacheck, selene, tree-sitter AST анализ, контрактные правила, разрешение кросс-файловых зависимостей, анализ цикломатической сложности, обнаружение паттернов крашей и уязвимостей, интеграционное тестирование lua54 со стабами движка X-Ray, сканирование секретов gitleaks.
Полный отчёт в doc/test-report.log.
